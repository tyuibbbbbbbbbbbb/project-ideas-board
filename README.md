# לוח הרעיונות - Project Ideas Board

אתר שבו כל אחד יכול לפרסם רעיון לפרויקט שהוא לא יכול/רוצה לפתח בעצמו,
ומפתחים אחרים יכולים לגלוש, להצביע (לייק/דיסלייק) ולאמץ ("יאללה עלי!") רעיונות.

- **בלי סיסמה ובלי הרשמה** - פשוט בוחרים שם תצוגה ומתחילים.
- רעיונות ממוינים לפי ניקוד (לייקים פחות דיסלייקים) - הכי מוצבע למעלה.
- רעיון עם יותר מ-10 דיסלייקים עובר אוטומטית לסטטוס "בבדיקה".
- כשמישהו לוקח רעיון, הכרטיס משנה צבע (ירוק) ומוצג "נלקח ע"י ...".
- מי שאימץ רעיון יכול לסמן אותו כ"בוצע" כשהוא מסיים.
- אפשר לשמור רעיונות "לעצמי" (רשימה פרטית, נשמרת רק בדפדפן שלך).

## איך זה בנוי

| חלק | איפה |
|---|---|
| האתר (frontend) | **GitHub Pages** - קבצי HTML/CSS/JS סטטיים |
| הנתונים | **Firestore** (Google Cloud) - מסד נתונים בזמן אמת |
| זיהוי משתמשים | **Firebase Anonymous Auth** - מזהה אוטומטי לכל דפדפן, בלי סיסמה |

האתר מדבר ישירות מול Firestore (דומיין של Google). אין שרת ביניים לתחזק.
האבטחה נשלטת ע"י כללי Firestore (`firestore.rules`).

## הקמה (חד פעמית, ~5 דקות)

1. צרו פרויקט ב-<https://console.firebase.google.com/> (חינם).
2. **Build → Authentication → Get started →** הפעילו את הספק **Anonymous**.
3. **Build → Firestore Database → Create database →** בחרו **Production mode** ומיקום.
4. העתיקו את כללי האבטחה מהקובץ `firestore.rules` אל לשונית **Rules** ב-Firestore ולחצו **Publish**.
5. **Project settings → Your apps → Web (`</>`)** → העתיקו את אובייקט ה-`firebaseConfig`
   והדביקו אותו בקובץ `firebase-config.js`.
6. `git push` - האתר יתעדכן אוטומטית ב-GitHub Pages.

## הרצה מקומית

```
npm run dev
```

ואז פתחו <http://localhost:5500>.

## מבנה הפרויקט

| קובץ | תיאור |
|---|---|
| `index.html` | מבנה הדף |
| `style.css` | עיצוב (ערכת נושא כהה עם זהב) |
| `firebase-config.js` | הגדרות חיבור ל-Firebase (יש למלא) |
| `app.js` | כל הלוגיקה - שם תצוגה, פרסום, הצבעה, אימוץ, שמירה |
| `firestore.rules` | כללי אבטחה של מסד הנתונים |
